Logo Anatomy
Two brand elements distinguish the Shifaa logo from a pure wordmark — both in Electric Sky #0088CC.
① Teardrop / Heart mark
Acts as the dot above the i. A stylized teardrop representing care, health, and the drop of life. Never used in isolation.
② ECG / Pulse wave
An ECG heartbeat pulse spanning the full width of the wordmark. Represents monitoring, health tracking, and the rhythm of care. The horizontal lines extend into the negative space on both sides.
Clear Space & Usage Rules

Minimum clear space = height of the teardrop mark on all 4 sides. The dashed box is the exclusion zone — no text, graphics, or other elements inside it.

✓
Use on Navy #003E78, Sky #0088CC, or dark #080C14 backgrounds only.
✓
Maintain minimum height of 28px in digital, 15mm in print.
✕
Never place on white, light grey, or photographic backgrounds — the white letterforms will be invisible.
✕
Never recolor, reorder, or separate the teardrop/wave elements from the wordmark.
✕
Never use the wave or teardrop in isolation — they only function as part of the full wordmark.

BrandBook

Do's & Don'ts

Brand usage rules across every touchpoint. Each rule is categorized and paired — what to do and why, what to avoid and why it matters.

Colors
✓
Use Electric Sky #0088CC for all primary CTAs and interactive links.
It's the visual anchor of the brand. All primary actions must be instantly recognizable.
✕
Never use Gold #C8952A or Navy #003E78 as the primary CTA color.
Gold is reserved for premium/enterprise tiers. Navy is for depth — not primary action.
✕
Never use Error red #DC2626 for decorative purposes.
Red carries the weight of urgency. Using it casually trains users to ignore real errors.
✕
Never use Hint #8FA5BC for body text or any content that must be readable.
It fails WCAG 2.1 AA contrast on white. Use Description #4A6380 as the minimum for body content.
Logo
✓
Use only approved logo files from the Shifaa brand team.
The wordmark has precise proportions built into the SVG. Never recreate or redraw it — always use the approved file.
✓
Maintain the minimum clear space equal to the cap-height of "S" on all four sides.
Clear space protects legibility. Crowded logos read as unprofessional, especially at small sizes.
✕
Never place the logo on photographic backgrounds or unapproved colors.
Approved backgrounds: Sky · Navy · White · Pearl. Everything else requires brand team sign-off.
✕
Never modify, rotate, recolor, animate, or distort the wordmark, teardrop, or ECG wave.
Brand equity is built on consistency. Every deviation erodes recognition over time.
Typography
✓
Use SF Pro for all English interfaces. Use Almarai (weights 300/400/700/800) for all Arabic interfaces.
These typefaces were chosen for legibility, cultural authenticity, and bilingual visual harmony.
✓
Use DM Mono for all numeric data, IDs, codes, timestamps, and technical labels.
Monospace aligns numbers into clean columns and signals "this is data, not prose."
✕
Never substitute generic fonts (Arial, Roboto, Helvetica Neue) for SF Pro or Almarai.
Generic fonts undermine the brand's premium positioning and bilingual consistency.
Spacing & Layout
✓
Apply the 4px spacing grid to all padding, margin, and gap values.
Consistent spacing creates visual rhythm. Valid values: 4 · 8 · 12 · 16 · 20 · 24 · 28 · 32 · 40 · 48 · 56 · 64.
✕
Never use off-grid spacing values (5px, 7px, 9px, 11px, 13px, 15px).
Off-grid values create visual inconsistency that's subtle but immediately felt as low quality.
Arabic & RTL
✓
Write Arabic copy natively in Kuwaiti-aware language — never machine-translate from English.
Machine translation destroys warmth and often produces medical errors. Every string must be reviewed by a native speaker.
✓
Mirror directional UI elements (arrows, chevrons, progress bars) in RTL layouts.
Arabic users read right-to-left. Non-mirrored directional icons feel broken and disrespectful.
✕
Never mix RTL and LTR text in the same paragraph without proper Unicode bidi control.
Without bidi control, mixed-direction text renders incorrectly and can garble medical information.

Changelog

Version History

A full record of every change made to the Shifaa Design System. Each version is tagged with its release date and a breakdown of what was added, changed, or fixed.

v1.7 Current
March 2026
Added: Changelog page — version history with dates and change breakdowns.
v1.6
March 2026
Changed: Form Groups — Radio group replaced native <input type="radio"> with the brand .radio-tab-group component matching the Tabs page.
v1.5
March 2026
Changed: Type Scale — Body size corrected from 15px → 14px (EN and AR rows).
Added: Progress Tracker (Atoms) — delivery-app style horizontal tracker with 4 step states (done, active, pending, error), pulse animation, and RTL support.
Added: Progress Tracker (Molecules) — 4 horizontal scenarios (appointment booking, prescription pipeline, lab result pipeline, payment error), vertical onboarding flow EN + AR, and breadcrumb navigation trails.
Improved: Avatars page — reorganized with sizes table, vertical color variant grid, and editable avatar (pencil badge) pattern with 4 size examples and props table.
Improved: Form Groups — expanded to full shadcn field reference: text, success, error, clearable, disabled, textarea, select, date, time, datetime-local, number, phone, email, password (toggle visibility), search, toggle group, checkbox group, radio group, and file upload. Full AR/RTL form added.
Improved: Voice & Tone — removed "don'ts" entirely. All message types now shown as actual notification cards (OS push, in-app screen card, alert banner) in EN + AR side by side.
Improved: Do's & Don'ts — redesigned as categorized rule cards (Colors, Logo, Typography, Spacing, Arabic & RTL). Each rule includes a "why it matters" explanation.
v1.4
March 2026
Improved: Sidebar — redesigned from flat text list to collapsible sections with colored icon tiles per category (📖 BrandBook, ⚛️ Atoms, 🧩 Molecules, 🏗️ Organisms, 🗂️ Templates, 📱 Pages). Sections collapse/expand on click.
Improved: Colors page — redesigned as inline rows: color swatch + name + role description + hex code, all on one line per token. Semantic colors show foreground and background swatch together.
Changed: Voice & Tone — condensed to a compact two-column reference table (message type / Do / Don't). 14 scenarios across Patient and Doctor apps.
Improved: Buttons page — replaced single-variant state demo with a full 8-row × 5-column state matrix. Every variant (Primary, Secondary, Ghost, Outline, Destructive, Navy, Gold, Default) shows all 5 states with exact CSS values.
Added: Steppers — horizontal (4-step, all-done, error-on-step-3) and vertical (booking flow EN + AR) with state props table. Added to Breadcrumb (Atoms) page.
Discontinued: UI Components file — brand guidelines file is now the single source of truth. shifaa-ui-components-v1.3.html was the last separate components file.
v1.3
March 2026
Added: Tabs (Molecules) — expanded from one variant to four: underline (default), pill/segment, checkbox multi-select, and radio single-select. All with EN + AR support and JS interactions.
Added: Breadcrumb Atom — anatomy breakdown: item states, separator variants (/, ›, ·), ellipsis collapse token, and props table.
Added: Breadcrumb Molecule — real in-app navigation trails for patient and doctor flows, collapsed path, badge-in-trail, and RTL Arabic variants with mirrored chevrons.
Added: Clearable inputs — input-wrap + input-clear (× button), shows on value, clears on click, RTL variant moves × to left side.
Improved: Voice & Tone — expanded from 3 to 18 scenarios. Patient app (10): booking, reminder, cancellation ×2, lab results, prescription, medication, empty states, onboarding, payment ×2, error. Doctor app (8): new patient, daily schedule, urgent alert, prescription, no-show, shift handover. All shown EN + AR with OS push notification card mockups.
Fixed: Buttons — removed all gradients and box-shadows to match shadcn flat style. Hover = opacity 90%. Focus = outline: 2px offset 2px.
Fixed: Alerts — corrected to shadcn exact pattern: white background base, foreground title, muted-fg description. Not colored text matching border.
Fixed: Inputs — removed box-shadow. Focus ring now uses 0 0 0 2px bg + 0 0 0 4px foreground (shadcn ring-offset pattern). Hover border changes to foreground.
Fixed: Badges — border-radius corrected to 9999px (pill), font-weight to 600, removed DM Mono font-family override.
v1.2
March 2026
Added: Full Atomic Design structure — BrandBook, Atoms, Molecules, Organisms, Templates, Pages as a single-page app with sidebar navigation.
Added: shadcn/ui aesthetic applied system-wide — white background, #FAFAFA sidebar, 1px #E4E4E7 borders, 6px border-radius, "Preview" toolbar containers, props tables.
Added: Organisms — Navigation bar, Data table (with patient records), Stats grid, Notification feed, Appointment card.
Added: Templates — Patient App layout (with bottom nav), Doctor App layout (with sidebar nav).
Added: Pages — Patient Home (full composition), Doctor Dashboard (full composition).
Added: UI Components file (shifaa-ui-components-v1.2.html) — Stripe-inspired Glassmorphism UI kit created as a separate companion file.
v1.1
March 2026
Fixed: Year references updated from 2025 → 2026 throughout all content (version badge, patient IDs, dates, captions).
Changed: Filename updated to include version number — shifaa-brand-guidelines-v1.1.html. All subsequent files follow this convention.
Improved: Voice & Tone — expanded from 2 scenarios to 8. Added: push notification reminder (OS card mockup), patient cancels, doctor cancels, lab results (with Don't: alarming language), prescription ready, medication reminder (OS card), empty states.
v1.0 Initial release
March 2026
Added: Brand identity — logo wordmark (white letterforms + Electric Sky teardrop + ECG wave), colors (Electric Sky #0088CC, Navy #003E78, Heritage Gold #C8952A, Soft Pearl #EEF3F9), semantic colors.
Added: Typography — SF Pro (English), Almarai weights 300/400/700/800 (Arabic), DM Mono (labels/data). Full bilingual EN + AR font color stack.
Added: Button system — 6 variants (Primary, Secondary, Ghost, Outline, Danger, Navy, Gold), 5 states, 4 sizes. All variants include Arabic/RTL rows.
Added: Spacing & grid — 4px grid system, all padding/margin/gap values documented as multiples of 4: 4 8 12 16 20 24 28 32 40 48 56 64.
Added: Icons — native OS emoji (Apple Color Emoji / Noto with Android detection via body.is-android). Healthcare icon set documented.
Added: Voice & Tone — 4 pillars (Clear, Warm, Confident, Grounded). Initial 2 scenarios: appointment confirmation and error message, EN + AR.
Added: Co-marketing guidelines — 4 rules for clinic, lab, and insurance partner materials.
Added: Do's & Don'ts — initial 14 rules across color, logo, typography, and Arabic usage.
